# 德州扑克内容消费 App MVP PRD

## 1. 背景与目标

面向海外华语德州扑克用户，建设一款以学习内容、工具训练、社区分享和课程付费为核心的 iOS + Android 原生 App。

产品目标：

- 让用户在一个 App 内完成德州扑克知识学习、视频观看、工具计算、牌谱复盘和课程购买。
- 通过「首页内容流 + 工具 + 课程 + 我的」形成清晰的消费与转化路径。
- 支持运营团队通过后台维护文章、视频、课程、直播预告/回放和冷启动社区内容。
- 首版不提供真钱牌桌、约局撮合、筹码兑换或下注交易能力。

## 2. 用户角色

| 角色 | 说明 | 核心诉求 |
|---|---|---|
| 游客 | 未登录用户 | 浏览公开内容、试看课程、了解工具能力 |
| 注册用户 | 已登录普通用户 | 关注作者、收藏内容、发布动态、使用工具、购买课程 |
| 课程购买用户 | 已购买单课的用户 | 学习课程章节、查看学习进度、复看已购课程 |
| 内容运营 | 后台小编/运营人员 | 上传和管理文章、视频、课程、直播、冷启动动态 |
| 审核运营 | 内容审核人员 | 审核社区动态、评论、举报和违规内容 |
| 管理员 | 后台管理员 | 管理用户、订单、内容状态、审核规则和操作日志 |

## 3. 功能范围

### 3.1 底部 Tab

底部固定 4 个 Tab：

- 首页
- 工具
- 课程
- 我的

首页内部频道：

- 关注
- 推荐
- 知识
- 社区
- 视频
- 直播

### 3.2 MVP 包含

- 首页多频道内容流。
- 知识文章浏览、详情、收藏、点赞、评论、举报。
- 中长视频浏览、播放、收藏、点赞、评论、观看进度。
- 社区动态发布、审核、信息流、评论、点赞、关注、举报。
- 直播频道的直播预告、直播中状态展示、回放入口和预约提醒。
- 工具页的胜率计算器、文本牌谱 AI 复盘、复盘历史。
- 课程页的课程列表、详情、章节、试看、单课购买、学习进度。
- 我的页的用户主页、资料编辑、关注/粉丝、收藏、已购课程、学习记录、复盘记录、设置。
- 运营后台的内容、课程、直播、社区冷启动内容、审核、用户、订单、操作日志管理。

### 3.3 MVP 不包含

- 实时牌桌、真钱游戏、下注交易、筹码兑换。
- 线下约局撮合。
- 实时推流、连麦和聊天室。
- 截图识别牌谱、视频自动复盘。
- 会员订阅。

## 4. 业务流程

### 4.1 内容消费流程

1. 用户进入首页。
2. 用户切换频道：关注、推荐、知识、社区、视频、直播。
3. 系统按频道返回对应内容流。
4. 用户点击内容进入详情。
5. 用户可点赞、收藏、评论、举报。
6. 未登录用户触发互动时，引导登录。

### 4.2 社区发布流程

1. 登录用户进入社区频道或我的页入口。
2. 用户发布文字、图片或牌谱摘要。
3. 系统进行敏感词和基础格式校验。
4. 动态进入 `pending_review` 状态。
5. 审核通过后进入 `published` 状态并展示。
6. 审核拒绝后进入 `rejected` 状态，并向用户展示拒绝原因。

### 4.3 课程购买流程

1. 用户进入课程列表。
2. 用户查看课程详情和试看章节。
3. 用户点击购买。
4. iOS 调用 Apple IAP，Android 调用 Google Play Billing。
5. 后端校验支付凭证。
6. 校验成功后创建或更新订单，并发放课程权益。
7. 用户进入课程章节学习，系统记录学习进度。

### 4.4 工具使用流程

1. 用户进入工具 Tab。
2. 用户选择胜率计算器或 AI 复盘。
3. 胜率计算器：输入手牌、公共牌、玩家人数，系统返回胜率/平局率/败率。
4. AI 复盘：用户粘贴文本牌谱，系统解析牌谱并返回局面摘要、行动线分析、关键错误和替代打法。
5. AI 复盘结果保存到用户复盘历史。

## 5. 字段与规则

### 5.1 内容通用字段

| 字段 | 说明 |
|---|---|
| 标题 | 文章、视频、课程、直播内容标题 |
| 摘要 | 内容列表展示摘要 |
| 封面图 | 列表和详情页封面 |
| 内容类型 | article、video、post、course、live |
| 分类 | 知识、视频、直播等业务分类 |
| 标签 | 多个标签，用于搜索和推荐 |
| 作者 | 内容作者或运营账号 |
| 状态 | draft、pending_review、approved、rejected、published、offline、deleted |
| 推荐权重 | 运营控制推荐频道排序 |
| 互动计数 | 点赞、收藏、评论、分享、浏览/播放 |

### 5.2 课程字段

| 字段 | 说明 |
|---|---|
| 课程名称 | 课程主标题 |
| 讲师 | 课程讲师或运营账号 |
| 价格 | 单课购买价格 |
| 试看章节 | 是否允许试看 |
| 章节列表 | 课程章节、视频、时长、排序 |
| 权益状态 | 是否已购买、是否可观看 |

### 5.3 风控规则

- 社区动态和评论默认先审后发。
- 禁止发布真钱牌桌、下注、筹码兑换、约局撮合、外部支付导流等内容。
- 被封禁用户不可发布动态、评论或发起举报。
- 所有审核、下架、封禁、退款、撤权动作必须记录操作日志。

```text

→ published

```

## 8. 异常场景

- 未登录用户触发互动、发布、购买、复盘时，引导登录。
- 课程支付成功但权益未发放时，用户可手动恢复购买，后端按凭证幂等补发。
- 视频播放失败时展示重试、切换网络提示和错误码。
- AI 复盘解析失败时提示用户检查牌谱格式，并保留原始输入草稿。
- 审核拒绝时必须展示明确原因，不展示内部风控规则细节。
- 内容下架后，列表不可见；已收藏入口展示“内容已下架”。
- 用户被封禁后，不可发布、评论、点赞或关注。

## 9. 验收标准

- 底部 4 个 Tab 可正常切换，首页 6 个频道内容流可正常展示。
- 后台发布并上架文章、视频、课程、直播内容后，客户端可按状态展示。
- 社区动态和评论先进入审核，通过后才对其他用户可见。
- 单课购买完成后，用户可观看付费章节；退款后权益撤销。
- 胜率计算器可返回基础胜率、平局率和败率。
- 文本牌谱 AI 复盘可返回结构化分析，并保存历史记录。
- 用户主页、关注、粉丝、收藏、学习记录、复盘记录可正常访问。
- 所有敏感操作均有后台操作日志。

## 10. 假设项

- 首发语言为简体中文，目标用户为海外华语用户。
- 首版使用 React Native 同时交付 iOS 和 Android。
- 首版支付为单课购买，不做会员订阅。
- 直播首版只做直播频道、预告、状态和回放入口，不做实时互动直播系统。
- AI 复盘第一版只支持文本牌谱。
