# 德州扑克内容消费 App MVP UI/UX 说明

## 1. 页面清单

| 模块 | 页面 |
|---|---|
| 首页 | 关注频道、推荐频道、知识频道、社区频道、视频频道、直播频道 |
| 内容详情 | 文章详情、视频详情、社区动态详情、直播详情/回放详情 |
| 工具 | 工具首页、胜率计算器、AI 复盘输入页、AI 复盘结果页、复盘历史 |
| 课程 | 课程列表、课程详情、章节播放、购买确认、购买结果、学习记录 |
| 我的 | 我的首页、用户主页、编辑资料、关注列表、粉丝列表、收藏、已购课程、设置、举报/反馈 |
| 后台 | 内容管理、课程管理、直播管理、社区内容管理、审核队列、用户管理、订单管理、操作日志 |

## 2. 用户路径

- 内容消费路径：首页 → 切换频道 → 内容卡片 → 详情 → 点赞/收藏/评论/举报。
- 课程转化路径：首页推荐/课程 Tab → 课程详情 → 试看 → 购买 → 支付结果 → 章节学习。
- 工具路径：工具 Tab → 胜率计算器/AI 复盘 → 输入 → 结果 → 保存历史。
- 社区路径：首页社区频道 → 发布动态 → 等待审核 → 审核通过后展示。
- 个人路径：我的 → 用户主页/收藏/已购课程/学习记录/复盘记录。

## 3. 页面结构

### 3.1 底部 Tab

底部固定 4 个入口：

- 首页
- 工具
- 课程
- 我的

选中态使用高对比主色和图标填充，未选中态使用中性灰。Tab 文案必须保持短文本，避免换行。

### 3.2 首页

首页顶部为频道切换：

- 关注
- 推荐
- 知识
- 社区
- 视频
- 直播

频道切换固定在顶部，内容列表垂直滚动。推荐频道支持混合卡片，其他频道优先展示单一内容类型。

### 3.3 工具

工具首页以功能列表呈现：

- 胜率计算器
- AI 实战复盘
- 复盘历史

胜率计算器采用表单输入 + 结果面板结构。AI 复盘采用大文本输入区 + 示例格式 + 提交按钮 + 结果分段展示。

### 3.4 课程

课程列表展示课程封面、标题、讲师、价格、试看标识、学习人数。课程详情展示课程介绍、章节目录、试看入口、购买按钮和购买后学习入口。

### 3.5 我的

我的首页展示头像、昵称、简介、关注/粉丝、收藏、已购课程、学习记录、复盘记录、设置和反馈入口。

## 4. 组件规范

| 组件 | 说明 |
|---|---|
| 内容卡片 | 用于文章、视频、动态、直播、课程的列表展示 |
| 频道 Tab | 首页顶部频道切换 |
| 底部 Tab | App 一级导航 |
| 操作按钮 | 点赞、收藏、评论、分享、举报、关注 |
| 视频播放器 | 支持播放、暂停、进度、全屏、错误重试 |
| 审核状态标签 | 展示待审核、已拒绝、已下架等状态 |
| 课程章节列表 | 展示章节名、时长、试看、锁定状态 |
| 支付确认面板 | 展示课程名、价格、平台支付提示 |
| 工具输入表单 | 手牌、公共牌、玩家人数、文本牌谱输入 |
| 空态/错误态 | 用于无内容、无网络、加载失败、权限不足 |

## 5. 交互链路

- 未登录状态点击互动按钮时，弹出登录引导。
- 关注按钮点击后立即反馈，并在失败时回滚状态。
- 评论提交后进入待审核状态，展示“审核通过后可见”。
- 社区动态发布后进入待审核列表，用户本人可见状态。
- 视频播放中断时保留播放进度，恢复后从上次进度继续。
- 课程试看章节可直接播放，付费章节未购买时展示购买引导。
- AI 复盘提交后进入加载态，结果页按“局面摘要、行动分析、关键错误、改进建议”分段展示。

## 6. 状态与反馈

| 状态 | 表现 |
|---|---|
| 加载态 | 骨架屏或按钮 loading |
| 空态 | 明确说明当前没有内容，并提供返回/刷新/去关注等动作 |
| 错误态 | 展示错误原因、重试按钮和必要错误码 |
| 未登录态 | 展示登录引导，不隐藏可浏览的公开内容 |
| 待审核态 | 用户本人可见状态标签，其他用户不可见 |
| 已下架态 | 详情不可访问，展示“内容已下架” |
| 支付处理中 | 禁用重复点击，展示恢复购买入口 |

## 7. 表单校验

| 表单 | 校验规则 |
|---|---|
| 登录 | 邮箱或手机号必填，验证码/密码必填 |
| 编辑资料 | 昵称必填，头像格式合法，简介长度限制 |
| 发布动态 | 文本或图片至少一项，敏感词拦截，图片数量限制 |
| 评论 | 内容必填，敏感词拦截，长度限制 |
| 胜率计算器 | 手牌合法、公共牌不重复、玩家人数合法 |
| AI 复盘 | 文本牌谱必填，长度限制，格式异常可提交但需提示 |
| 后台内容 | 标题、分类、封面、正文/视频/章节等关键字段必填 |

## 8. 响应式与可访问性

- 客户端适配主流 iOS 与 Android 屏幕尺寸。
- 文字不得遮挡内容，按钮最小点击区域不低于 44px。
- 视频、课程、工具结果页需支持弱网重试。
- 关键操作按钮需有明确禁用态和提交中状态。
- 颜色对比度满足移动端阅读要求。

## 9. 验收标准

- 首页 6 个频道切换顺滑，内容类型与频道定位一致。
- 底部 Tab 位置稳定，切换后保留各 Tab 的滚动和页面状态。
- 内容详情页互动入口完整，未登录引导正确。
- 课程购买链路展示清晰，未购买和已购买状态差异明确。
- 工具页输入、错误、结果、历史记录链路完整。
- 社区发布、审核状态、拒绝原因、举报反馈可被用户理解。
