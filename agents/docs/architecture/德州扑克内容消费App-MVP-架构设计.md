# 德州扑克内容消费 App MVP 架构设计

## 1. 技术目标

- 使用 React Native + TypeScript 交付 iOS 与 Android 客户端。
- 使用统一后端 API 支撑内容、社区、课程、工具、支付、后台管理。
- 支持运营后台管理内容、课程、直播、审核、订单和用户。
- 支持后续扩展推荐算法、实时直播、会员订阅和更复杂训练工具。

## 2. 模块划分

| 模块 | 职责 |
|---|---|
| 移动客户端 | 首页、工具、课程、我的、内容详情、支付和互动 |
| 管理后台 | 内容管理、课程管理、直播管理、审核、用户、订单、日志 |
| API 服务 | 用户、内容、社区、课程、订单、工具、审核接口 |
| 内容服务 | 文章、视频、直播预告/回放、标签、搜索、推荐权重 |
| 社区服务 | 动态、评论、点赞、关注、举报、审核状态 |
| 课程服务 | 课程、章节、试看、学习进度、课程权益 |
| 支付服务 | Apple IAP / Google Play Billing 凭证校验、订单、退款撤权 |
| 工具服务 | 胜率计算、文本牌谱解析、AI 复盘、复盘历史 |
| 审核风控 | 敏感词、先审后发、举报处理、封禁、操作日志 |

## 3. 数据流

### 3.1 首页内容流

```text
App 首页频道
→ API 网关
→ 内容/社区/课程/直播聚合查询
→ 过滤未发布/不可见内容
→ 按频道和推荐权重排序
→ 返回分页结果
```

### 3.2 课程购买

```text
App 发起购买
→ Apple IAP / Google Play Billing
→ App 获取支付凭证
→ 后端校验凭证
→ 创建/更新订单
→ 发放课程权益
→ App 刷新权益并允许学习
```

### 3.3 AI 复盘

```text
用户提交文本牌谱
→ API 校验登录和限流
→ 牌谱解析
→ AI 分析
→ 保存复盘记录
→ 返回结构化结果
```

## 4. API 边界

- 客户端只通过后端 API 获取内容、课程权益、订单状态和复盘结果。
- 客户端不得直接信任本地支付成功状态，必须以后端凭证校验结果为准。
- 后台接口与客户端接口分离，后台接口必须要求运营/管理员权限。
- 审核、下架、封禁、退款、撤权等高风险接口必须写操作日志。

## 6. 权限与安全

- 未登录用户只能浏览公开内容和试看课程。
- 登录用户可互动、发布动态、使用工具、购买课程。
- 被限制或封禁用户不可发布、评论、点赞、关注或举报。
- 运营人员只能管理授权范围内的内容与审核任务。
- 管理员可管理用户、订单、审核配置和操作日志。
- 敏感词、举报、审核记录和封禁记录不得暴露内部判定细节给普通用户。
- 支付回调和凭证校验必须幂等。

## 7. 异常处理

- 内容不存在、已下架、无权限时返回明确错误码。
- 支付凭证重复提交时返回已有订单和权益状态。
- AI 服务超时或失败时保存失败状态，允许用户重试。
- 视频资源不可用时返回备用错误提示，不影响其他内容浏览。
- 审核服务异常时，用户发布内容进入待处理状态，不直接公开。

## 8. 日志与监控

需要记录：

- 登录、注册、注销、封禁。
- 内容发布、审核、上下架、删除。
- 课程购买、凭证校验、权益发放、退款、撤权。
- AI 复盘请求、失败、耗时和消耗。
- 举报、处理结果和操作人。
- 后台所有高风险操作日志。

核心监控指标：

- DAU、注册转化、内容点击率、视频播放完成率。
- 课程详情转化率、购买成功率、支付失败率。
- 社区发布量、审核通过率、举报量。
- AI 复盘成功率、平均耗时、失败原因分布。

## 9. 风险与取舍

- 德州扑克内容可能被应用商店识别为博彩相关，需要明确学习工具定位，避免真钱、撮合和筹码兑换功能。
- 单课购买属于 App 内数字内容，iOS 和 Android 需分别遵守平台内购规则。
- 直播首版只做内容频道，避免实时音视频系统拖慢 MVP。
- AI 复盘先做文本牌谱，避免 OCR/视频解析导致成本和准确率不可控。

## 10. 回滚方案

- 内容模块支持后台一键下架。
- 课程权益发放支持按订单撤权。
- 社区违规内容支持下架、隐藏和封禁用户。
- AI 复盘异常时可关闭入口或降级为“提交失败，请稍后重试”。
- 支付异常时可暂停购买按钮，保留恢复购买入口。
