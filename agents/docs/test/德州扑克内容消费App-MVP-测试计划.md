# 德州扑克内容消费 App MVP 测试计划

## 1. 测试目标

验证德州扑克内容消费 App MVP 的核心链路可用、权限正确、支付权益一致、内容审核有效，并覆盖 iOS 与 Android 的主要使用场景。

## 2. 测试范围

- 底部 Tab：首页、工具、课程、我的。
- 首页频道：关注、推荐、知识、社区、视频、直播。
- 内容详情：文章、视频、社区动态、直播/回放。
- 互动：点赞、收藏、评论、举报、关注。
- 工具：胜率计算器、文本牌谱 AI 复盘、复盘历史。
- 课程：课程列表、详情、试看、购买、权益、学习进度。
- 用户：登录、资料、主页、关注/粉丝、收藏、已购课程。
- 后台：内容管理、课程管理、直播管理、审核队列、订单、用户、操作日志。

## 4. 异常与边界

- 弱网或断网下，列表、视频、支付、AI 复盘必须给出可恢复提示。
- 支付成功但后端校验超时时，用户可触发恢复购买。
- 内容被下架后，收藏入口不可继续访问详情。
- 被封禁用户不可绕过客户端限制直接调用发布接口。
- 待审核内容仅作者本人和后台可见。
- AI 复盘接口需要限流，超过限制展示明确提示。

## 5. 回归测试

每次发布至少回归：

- 登录和用户状态。
- 首页 6 个频道内容展示。
- 文章/视频/社区/直播详情。
- 课程购买和权益校验。
- 胜率计算器和 AI 复盘。
- 审核通过、拒绝、下架、举报。
- 后台操作日志。

## 6. 验收标准

- 所有 P0/P1 用例通过。
- 支付、权益、审核、封禁相关链路无阻断缺陷。
- iOS 与 Android 主流程一致。
- 后台变更能在客户端按状态正确呈现。
- 所有高风险操作均能在操作日志中追溯。
